# Day 8 – 🐞 Bug Life Cycle Explained

Understanding the **Bug Life Cycle** is crucial for every QA Engineer.  
It ensures issues are tracked consistently and resolved efficiently. 👌

---

## 🔄 Typical Bug Life Cycle Stages

```text
New → Assigned → In Progress → Fixed → Retest → Closed
                         ↘ Reopen ↗
```

---

## 📝 Stage Descriptions

| Stage        | Description |
|-------------|-------------|
| **New**        | Tester reports a valid bug, logged into the tracking system (e.g., Jira). |
| **Assigned**   | The bug is assigned to a developer or team lead for investigation. |
| **In Progress**| Developer starts working on fixing the issue. |
| **Fixed**      | Developer marks the bug as fixed and pushes the changes. |
| **Retest**     | QA re-tests the fix to verify if the issue is resolved. |
| **Closed**     | QA marks the bug as closed if it works correctly. |
| **Reopen**     | If the issue persists after the fix, QA reopens the bug. |

---

## 🧠 QA Tip:
> A clean, well-documented bug life cycle avoids confusion and helps the team maintain high quality.  
> Use consistent severity/priority levels and always add **clear reproduction steps** 🧪✨
