# Contributing

Thanks for considering contributing!

## Workflow
1. Create a feature branch off `main`
2. Write tests for your changes
3. Ensure `pytest -q` passes locally
4. Open a PR and describe the change, context, and test evidence

## Commit Style
- Use clear, imperative messages: `feat: add x`, `fix: correct y`
- Reference issues with `#123` when relevant

## Code Style
- Keep it simple and tested
- Prefer small, focused PRs
